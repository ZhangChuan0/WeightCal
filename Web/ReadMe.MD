# 体重智慧上传 - 项目需求文档

## 1. 项目概述
开发一个前后端协同的体重数据采集与可视化系统。用户通过电脑双摄像头完成**身份识别**与**体重秤数字OCR识别**，经手势确认后上传数据，并在前端查看历史体重趋势图。

## 2. 技术栈
- **前端**：Vue3 + Vite + TypeScript + Pinia + ECharts + face-api.js + MediaPipe Hands
- **后端**：SpringBoot 2.7+ / 3.x + MyBatis-Plus + Tess4J (OCR)
- **数据库**：MySQL 8.0
- **运行要求**：HTTPS 环境（摄像头权限），Java 17+，Node 16+

## 3. 功能模块与需求

### 3.1 摄像头管理
- [ ] 支持同时启用两个摄像头（如：内置摄像头用于人脸，外接摄像头对准体重秤）
- [ ] 提供设备选择下拉框，用户可手动切换摄像头设备
- [ ] 视频流实时预览，无卡顿

### 3.2 身份识别（人脸）
- [ ] 加载 face-api.js 模型（SSD MobileNet + FaceRecognition）
- [ ] 每隔 500ms 检测视频中是否有人脸
- [ ] 提取人脸特征向量（128维数组）
- [ ] 将特征向量发送后端进行用户匹配
- [ ] 匹配成功后显示用户姓名，锁定身份直到测量结束
- [ ] 支持新用户注册（上传人脸照片并提取特征存储）

### 3.3 体重秤 OCR 识别
- [ ] 切换到体重秤摄像头后，自动定时截图（每1秒）
- [ ] 将截图上传后端进行 OCR 识别
- [ ] 后端返回识别出的体重数值（kg）
- [ ] 前端连续采集3个稳定值（差值 < 0.1）后视为有效读数
- [ ] 停止截图轮询，进入确认状态

### 3.4 手势确认
- [ ] 弹窗显示当前用户、体重读数，提示“请做OK手势确认，拒绝请比×”
- [ ] 复用摄像头1流，使用 MediaPipe Hands 检测手势
- [ ] 识别到 OK 手势（拇指与食指捏合，其余三指伸直）且稳定8帧后触发确认
- [ ] 支持手势拒绝（例如五指张开）或超时自动取消
- [ ] 确认后调用上传接口，取消则返回体重采集状态

### 3.5 数据上传与存储
- [ ] POST 请求上传体重记录（用户ID、体重值、测量时间、可选的截图路径）
- [ ] 后端校验数据合法性（体重 > 0）
- [ ] 记录存入 `weight_record` 表
- [ ] 返回上传结果及最新记录

### 3.6 可视化图表
- [ ] 获取当前用户所有历史体重记录（GET 接口）
- [ ] 使用 ECharts 绘制折线图，X轴为时间，Y轴为体重
- [ ] 支持鼠标悬停查看详情
- [ ] 图表在每次上传成功后自动刷新

### 3.7 异常处理与状态机
- [ ] 定义明确的流程状态：`IDLE -> RECOGNIZING -> RECOGNIZED -> MEASURING -> CONFIRMING -> COMPLETED`
- [ ] 人脸识别失败或无权限时给予用户提示
- [ ] OCR 识别失败时重试3次，之后提示手动输入
- [ ] 摄像头断开时自动恢复或提示重新连接
- [ ] 手势超时自动取消，不阻塞流程

## 4. 数据库设计

### user 表
| 字段             | 类型         | 说明                     |
| ---------------- | ------------ | ------------------------ |
| id               | BIGINT       | 主键                     |
| name             | VARCHAR(50)  | 用户姓名                  |
| face_descriptor  | TEXT         | 人脸特征向量 JSON 数组    |
| created_at       | DATETIME     | 创建时间                  |

### weight_record 表
| 字段              | 类型           | 说明                     |
| ----------------- | -------------- | ------------------------ |
| id                | BIGINT         | 主键                     |
| user_id           | BIGINT         | 外键关联用户              |
| weight            | DECIMAL(5,2)   | 体重（kg）               |
| measurement_time  | DATETIME       | 测量时间                  |
| image_path        | VARCHAR(255)   | 体重秤截图保存路径（可选） |

## 5. API 接口设计

### 5.1 用户管理
- `POST /api/users`  
  描述：注册新用户  
  请求体：`{ "name": "张三", "faceDescriptor": [0.12, -0.34, ...] }`  
  响应：`{ "userId": 1, "name": "张三" }`

### 5.2 人脸识别匹配
- `POST /api/recognize`  
  描述：上传人脸特征向量，返回匹配用户  
  请求体：`{ "descriptor": [0.12, -0.34, ...] }`  
  响应成功：`{ "userId": 1, "name": "张三" }`  
  响应失败：`{ "userId": null, "message": "未匹配到用户" }`

### 5.3 OCR 识别体重
- `POST /api/ocr/weight`  
  描述：上传体重秤截图，返回识别数字  
  请求：`multipart/form-data`，字段 `image`  
  响应成功：`{ "weight": 70.5 }`  
  响应失败：`{ "weight": null, "message": "无法识别数字" }`

### 5.4 体重记录操作
- `POST /api/weights`  
  描述：新增体重记录  
  请求体：`{ "userId": 1, "weight": 70.5 }`  
  响应：`{ "id": 1, "userId": 1, "weight": 70.5, "time": "2026-06-08T10:20:30" }`

- `GET /api/weights?userId=1`  
  描述：获取用户所有体重记录  
  响应：`[{ "id":1, "weight":70.5, "time":"..." }, ...]`  
  排序：按测量时间升序

## 6. 前端状态管理（Pinia）
- **useUserStore**：存储当前识别到的用户 ID、姓名
- **useMeasureStore**：存储测量状态（'idle' | 'recognizing' | 'measuring' | 'confirming' | 'completed'）、体重采样数组、稳定读数
- **useDeviceStore**：存储两个摄像头的设备 ID

## 7. 开发步骤（建议迭代计划）

### 第一阶段：基础环境搭建
1. 初始化 Vue3 + SpringBoot 项目，配置依赖
2. 建立数据库表，完成 MyBatis 映射
3. 实现用户注册和查询接口

### 第二阶段：双摄像头与流程控制
4. 实现双摄像头选择和视频流展示
5. 搭建前端状态机，实现各状态切换逻辑

### 第三阶段：身份识别闭环
6. 集成 face-api.js，完成人脸检测与特征提取
7. 实现后端 `/api/recognize` 匹配逻辑
8. 联调身份识别全过程

### 第四阶段：体重 OCR 识别
9. 后端集成 Tess4J，编写图像预处理和数字识别
10. 前端实现体重秤摄像头定时截图上传
11. 实现稳定读数判断逻辑

### 第五阶段：手势确认与数据上传
12. 集成 MediaPipe Hands，实现 OK 手势识别
13. 开发确认弹窗，手势触发上传
14. 实现 `/api/weights` 接口并联调

### 第六阶段：可视化与完善
15. 实现历史记录查询和 ECharts 折线图
16. 异常处理完善、UI 优化
17. 全流程测试、文档编写

## 8. 注意事项
- 所有网络请求需携带必要错误处理提示，如网络断开、识别超时等
- OCR 图片建议先进行灰度化、二值化预处理，并限制识别字符集为 `0123456789.`
- 人脸特征向量存储时建议进行加密（可选）
- 手势确认阶段需提供清晰的视觉引导（如屏幕提示文字、边框高亮）
